# 环境文档

本项目（agent-learn）的运行环境与 API 配置说明。

## 一、Python 环境

- **Conda 环境名**：`agent`
- **Python 版本**：3.11.15
- **解释器路径**：`/data/ghs/miniconda3/envs/agent/bin/python`

激活方式：

```bash
conda activate agent
# 或直接调用绝对路径
/data/ghs/miniconda3/envs/agent/bin/python your_script.py
```

## 三、API 配置

项目根目录 `.env`（路径：`/data/ghs/agent-learn/.env`）：

```dotenv
OPENAI_API_KEY="sk-xxxxxxxxxxxxxxxxxxxx"
OPENAI_API_BASE="https://api.deepseek.com/v1"
```

- 模型接入的是 **DeepSeek** 的 OpenAI 兼容接口。
- 可用模型名：`deepseek-chat`、`deepseek-reasoner`（**不要写成 `deepseekv3`**，会报 `Model Not Exist`）。
- `OPENAI_API_BASE` 是旧变量名；新版 `openai` SDK 推荐 `OPENAI_BASE_URL`。代码中建议显式传入 `base_url` 以兼容两者。

## 四、最小可用示例

```python
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

load_dotenv()

model = ChatOpenAI(
    model="deepseek-chat",
    temperature=0.7,
    base_url=os.getenv("OPENAI_API_BASE") or os.getenv("OPENAI_BASE_URL"),
    api_key=os.getenv("OPENAI_API_KEY"),
)

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个资深的{domain}专家。"),
    ("human", "请解释一下什么是{topic}？"),
])

chain = prompt | model | StrOutputParser()
print(chain.invoke({"domain": "AI", "topic": "Transformer"}))
```

## 五、常见坑

1. **`Model Not Exist`**：模型名不对，DeepSeek 要用 `deepseek-chat`。
2. **`response_format type is unavailable`**：DeepSeek 当前不支持 `json_schema` 形式的结构化输出。使用 `with_structured_output(Schema, method="function_calling")` 走 function calling 模式。
3. **`.env` 未被读取**：`load_dotenv()` 默认只在当前工作目录向上查找。子目录脚本可显式传路径：
   ```python
   load_dotenv(os.path.join(os.path.dirname(__file__), "..", ".env"))
   ```
4. **`ModuleNotFoundError: dotenv`**：说明用的是系统/其他 Python，切到 `agent` 环境。
